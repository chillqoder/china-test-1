# 3D Solar System — Development Prompt

## Role

You are an **expert frontend developer and creative technologist** specializing in 3D web graphics, interactive simulations, and visually rich browser experiences. You have deep knowledge of 3D rendering pipelines, scene graphs, lighting models, and animation loops. You write clean, well-structured, and thoroughly commented code. Your goal is to produce a production-quality, visually stunning result that feels like a real interactive planetarium.

---

## Overview

Build an interactive **3D Solar System** simulation using **HTML, CSS, and JavaScript** as three separate files. The project must run directly from the filesystem by opening `index.html` in a browser — no local server required.

---

## File Structure

```
solar-system/
├── index.html
├── style.css
└── main.js
```

- `index.html` — markup and script/link references only
- `style.css` — all visual styling, layout, and UI theming
- `main.js` — all 3D logic, animation, and interactivity

---

## Planets & Celestial Bodies

Include the following objects, each with distinct size, color, orbital radius, and rotation/revolution speed:

| Body | Notes |
|---|---|
| ☀️ **Sun** | Central glowing star, largest object, emits light |
| 🪨 **Mercury** | Smallest, closest orbit, fast revolution |
| 🌍 **Venus** | Slightly smaller than Earth, slow rotation |
| 🌎 **Earth** | Recognizable blue-green appearance |
| 🔴 **Mars** | Reddish tone, slightly smaller than Earth |
| 🟠 **Jupiter** | Largest planet, visible band stripes |
| 🪐 **Saturn** | Prominent ring system around it |
| 🔵 **Uranus** | Pale blue-green, tilted axis |
| 💙 **Neptune** | Deep blue, outermost planet |

> Pluto and asteroid belt are optional bonus additions.

## Visual Style

- **Dark space theme** — deep black or very dark navy background
- **Starfield** — dense static or slowly twinkling star backdrop
- **Glowing Sun** — radiant bloom/glow effect at the center
- **Planet textures or shading** — use realistic colors and subtle gradients or procedural shading to differentiate planets; texture maps are encouraged if supported by the chosen library
- **Saturn's rings** — rendered as a flat disc around the planet
- **Smooth orbital paths** — faint elliptical or circular lines showing each planet's orbit
- **Ambient + point lighting** — light should emanate from the Sun and illuminate planet surfaces

---

## Interactivity

- **Orbit controls** — mouse drag to rotate the scene, scroll to zoom in/out
- **Click on a planet** — display a small info tooltip or label with planet name and basic stats (diameter, distance from Sun, orbital period)
- **Speed control** — a UI slider or buttons to speed up / pause / slow down the simulation
- **Toggle orbits** — button to show/hide orbital path lines
- **Camera reset** — button to return to the default top-down or angled overview

---

## Technical Requirements

- All three files must be **separate** (`index.html`, `style.css`, `main.js`)
- The simulation must **open directly in a browser** without any build step or local server
- **You MUST use a 3D rendering library** — implementing raw WebGL from scratch is forbidden. Load it via a CDN `<script>` tag inside `index.html`. The specific library is your choice, but it must provide a proper 3D scene graph, camera system, and lighting model. A plain 2D canvas approach is not acceptable.
- If the chosen library supports additional plugins (e.g. orbit controls, post-processing bloom), load those via CDN as well
- Animations must use a **render loop** driven by the library's own update cycle
- Code should be clean, commented, and modular

---

## UI / Layout

- **Full-screen canvas** occupying 100% of the viewport
- Minimal HUD overlay in a corner: speed slider, toggle buttons, planet labels on/off
- UI elements should have a **semi-transparent dark panel** style to not distract from the scene
- Font: monospace or futuristic sans-serif, light color on dark background
